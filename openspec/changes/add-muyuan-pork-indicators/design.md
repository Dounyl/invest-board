## Context

本变更围绕一个明确的投资动作：分三段判断牧原股份和猪肉养殖板块是否值得加仓。当前项目已有数据流水线和静态看板方向，但如果一次性纳入过多行业指标，会增加分析噪音。

设计上需要把数据、指标和展示范围控制在“能支持决策”的最小集合：情绪低点、行业拐点、板块情绪确认，以及头部猪企的成本与现金安全垫。

## Goals / Non-Goals

**Goals:**

- 建立三段加仓决策框架对应的数据模型。
- 只保留必要数据：牧原股价、全国能繁母猪存栏、全国生猪出场价格、头部猪企股价表现、头部猪企成本、头部猪企现金状态。
- 明确头部猪企横向对比范围，用于判断成本优势和资金安全。
- 为后续看板设计 Markdown 文档预留结构，但本阶段不实现前端页面。
- 保持数据源权威优先，避免依赖不可追溯或不可复核的来源。

**Non-Goals:**

- 不做完整猪周期研究库。
- 不纳入商品猪销售均价、生猪出栏量、猪肉产量、屠宰量、饲料价格、猪粮比价、猪料比价、销售收入等第一版非必要指标。
- 不在本阶段实现 Vue 页面、ECharts 图表或自动化采集脚本。
- 不给出买卖建议，只提供决策所需的数据观察框架。

## Decisions

### Decision 1: 用三段加仓动作组织看板，而不是按数据来源组织

Rationale: 用户的目标不是浏览所有行业数据，而是判断三次投入时机。看板应按“情绪低点、周期拐点、情绪确认”组织信息，使每个模块都对应一个具体动作。

Alternatives considered:
- 按公司、行业、财务三类组织：结构常见，但用户需要自行把数据映射回加仓动作。
- 按数据源组织：便于开发，但不利于投资判断。

### Decision 2: 行业拐点使用全国能繁母猪存栏，并用全国生猪出场价格做确认

Rationale: 用户明确认为拐点和母猪存栏量有关。能繁母猪存栏是供给周期的核心领先指标，但只看存栏容易过早判断拐点；全国生猪出场价格作为价格确认指标，可以验证供给变化是否开始传导到市场价格。

全国生猪出场价格第一来源采用国家发改委价格监测中心，因为其发布频率高于农业农村部月度专题数据；农业农村部数据作为月度复核来源。

Alternatives considered:
- 同时纳入生猪出栏、猪肉产量、屠宰量：信息更完整，但会把 MVP 变成行业数据库。
- 纳入饲料价格、猪粮比价、猪料比价：有助于利润判断，但会增加解释负担，第一版暂不纳入。

### Decision 3: 头部猪企名单第一版固定，只比较成本和现金状态

Rationale: 用户关心个股能否穿越周期，关键是成本位置和资金健康。第一版对比指标应服务“谁更能扛低谷”，而不是覆盖所有财务维度。

First-version companies:
- 牧原股份
- 温氏股份
- 新希望
- 天邦食品
- 巨星农牧
- 唐人神
- 大北农
- 正邦科技

Core comparison fields:
- 公司披露成本区间
- 货币资金
- 短期有息债务或一年内到期债务
- 经营现金流
- 资产负债率

Alternatives considered:
- 纳入营收、销量、利润率、产能：会增加解释负担，且与“成本和现金安全”关系不够直接。
- 允许通过配置增减公司名单：灵活性更高，但第一版会增加维护和口径对齐成本。
- 将现金安全转换为“安全、观察、紧张”标签：可读性更强，但阈值容易引入主观判断，第一版先展示原始财务指标。

### Decision 4: 股票行情不使用第三方数据源

Rationale: 股价数据需要可复核、权威和尽量少争议。第一版只使用交易所页面或授权行情源，不接入第三方行情数据。

Alternatives considered:
- 使用可复核的第三方行情源：接入便利，但来源权威性和长期稳定性不符合本阶段约束。

### Decision 5: 看板设计作为独立 Markdown 文档在下一阶段产出

Rationale: 当前阶段先确定数据边界和决策框架。看板设计需要进一步明确布局、字段、阈值展示和 JSON 契约，适合作为后续单独文档推进。

Suggested future artifact:
- `openspec/changes/add-muyuan-pork-indicators/dashboard-design.md`

Alternatives considered:
- 直接在 `design.md` 内完成看板设计：会让当前技术设计混入过多页面细节。
- 直接实现前端：当前数据契约和指标规格尚未完成，容易返工。

## Risks / Trade-offs

- [Risk] 只用能繁母猪存栏判断行业拐点可能过于简化 -> Mitigation: 增加全国生猪出场价格作为价格确认指标，仍不扩展到完整行业数据库。
- [Risk] 头部猪企成本口径可能不一致 -> Mitigation: 数据中保留来源、披露口径和报告期，不强行混成单一绝对排名。
- [Risk] 股票行情官方或授权源可程序化获取难度较高 -> Mitigation: 将行情源封装为独立数据适配层，第一版不使用第三方行情数据源。
- [Risk] 财务数据更新频率低于行情数据 -> Mitigation: 看板明确每类数据的更新时间，不把低频财务数据伪装成实时信号。

## Migration Plan

1. 先完成 OpenSpec specs，明确指标、数据源、输出契约和非目标范围。
2. 下一阶段新增看板设计 Markdown，描述三段加仓面板和头部猪企安全垫模块。
3. 实现时先生成静态 JSON 样例，再接真实数据采集。
4. 若需要回滚，删除本变更新增的 OpenSpec 文件和后续配置、脚本、SQL、导出与看板入口即可，不影响现有数据目录。
