## Why

当前项目需要把猪周期分析从“尽可能多的数据”收敛为服务三段加仓决策的最小指标集合，降低分析压力并提升看板的行动性。

本变更用于建立牧原股份与头部猪企的核心跟踪能力，帮助判断情绪低点、行业拐点和板块情绪确认，同时避免第一版变成完整行业数据库。

## What Changes

- 新增面向三段加仓策略的指标范围：情绪低点、行业拐点、板块情绪确认、头部猪企安全垫。
- 将第一版数据范围限制为必要指标：牧原股价、全国能繁母猪存栏、全国生猪出场价格、头部猪企股价表现、头部猪企成本水平、头部猪企现金状态。
- 明确不纳入第一版的噪音指标：商品猪销售均价、生猪出栏量、猪肉产量、屠宰量、饲料价格、猪粮比价、猪料比价、销售收入等。
- 使用权威数据源优先原则，优先采用交易所或授权行情源、巨潮资讯公告、农业农村部、国家统计局等来源，不使用第三方行情数据源。
- 看板设计先作为后续阶段的 Markdown 设计文档产出，不在本阶段直接实现页面。

## Capabilities

### New Capabilities

- `pork-equity-decision-indicators`: 定义牧原股份与头部猪企三段加仓决策所需的核心数据、来源边界和看板信息范围。

### Modified Capabilities

无。

## Impact

- 影响代码范围：`config/`、`scripts/`、`sql/`、`web/`。
- 影响文档范围：`openspec/changes/add-muyuan-pork-indicators/`，以及后续新增的看板设计 Markdown。
- 数据依赖：股票行情、上市公司公告、农业农村部、国家统计局、国家发改委价格监测中心等相关行业数据。
- 系统影响：后续会影响数据采集、清洗、导出 JSON 和静态看板展示范围。
- 回滚方式：删除本变更新增的 OpenSpec 文件及后续配置、脚本、SQL、导出和看板入口即可，不影响既有 raw 数据历史。
