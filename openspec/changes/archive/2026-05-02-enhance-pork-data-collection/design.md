## Context

现有猪肉权益看板以官方或授权数据为主，已经包含牧原股价、官方能繁母猪存栏、全国生猪出场价格、头部猪企股价表现和财务现金状态。实际使用中发现两个缺口：官方能繁母猪存栏频率偏低，难以解释短期猪价变化；板块情绪和公司财务压力缺少可追溯、可读性强的辅助指标。

本变更跨越配置、采集、转换、导出、SQL 检查和 Vue/ECharts 展示，重点是把“新增数据收集”做成可复现、可标注、可回滚的管线能力。

## Goals / Non-Goals

**Goals:**

- 为第三方样本能繁母猪/产能监测建立独立指标、来源、raw 路径和导出字段。
- 明确第三方样本监测只能作为辅助高频口径，不能与官方能繁母猪存栏绝对量混线。
- 为板块情绪确认导出公司级今年以来股价序列、近 20 交易日涨跌幅、权重和加权情绪值。
- 改善页面可读性：图表拆分、公司走势交互、金额中文单位展示、移除无决策价值面板。
- 保留 raw 只追加和真实数据优先原则。

**Non-Goals:**

- 不购买或接入付费 Mysteel、卓创资讯、涌益咨询 API。
- 不把第三方样本数据当作官方统计替代值。
- 不生成“现金安全/危险”等主观标签。
- 不引入登录、写接口或用户可配置股票池。

## Decisions

1. **第三方样本监测使用独立指标**

   新增 `third_party_sow_capacity_monitoring`，单位为 percent，原始路径为 `data/raw/pork/third_party_sow_capacity_monitoring/`。这样可以在数据模型层阻止它与 `breeding_sow_inventory` 的万头绝对量混用。

   备选方案是把第三方数据作为官方存栏的补点，但这会造成来源、样本范围、单位和统计口径混乱，因此不采用。

2. **公开转载数据先以种子采集器落 raw**

   公开可验证的 2026 年 Mysteel、卓创资讯、涌益咨询数据主要是样本环比，不是完整 API。采集器使用带 source_url 的种子记录写入 raw，并按 provider+period 去重，后续可替换为真实 API 或结构化网页适配器。

   备选方案是让前端硬编码这些数据，但这会绕过 raw 追加、导出和检查链路，因此不采用。

3. **板块情绪按固定权重计算**

   继续使用固定头部猪企名单。权重写入公司配置：牧原 35%、温氏 20%、新希望 15%、其余五家公司各 6%。综合情绪值为标准化权重乘以近 20 个交易日涨跌幅之和，并输出覆盖权重和上涨样本权重。

   备选方案是按市值动态权重计算，但这需要新增市值数据源和复权口径，超出本次范围。

4. **财务仍展示原始指标，不新增安全标签**

   头部猪企表格将金额格式化为万/亿，便于中文阅读；资产负债率、货币资金、短期有息债务和经营现金流仍作为原始披露指标展示。现金覆盖短债等派生压力指标可作为后续变更，但本变更不生成安全标签。

## Risks / Trade-offs

- [Risk] 第三方样本数据来自公开转载，可能滞后或口径不完整 → 用 `source_url`、`provider`、`sample_scope`、`third_party_sample` 标记，并通过 SQL check 要求口径字段存在。
- [Risk] 种子采集器需要人工维护新月份数据 → 使用独立 adapter 封装，后续可替换为真实公开页面解析或授权 API。
- [Risk] 固定情绪权重带有主观判断 → 在页面写明权重和公式，输出覆盖权重，不隐藏计算口径。
- [Risk] 图表增多导致页面过长 → 将行业拐点拆成多 panel，减少辅助线和说明性面板，优先保留决策相关信息。
